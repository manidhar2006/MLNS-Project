# HGAT Pathways Notebook Implementation Guide

This README explains how HGAT with KEGG pathway context is implemented in HGAT_Pathways.ipynb.

## Goal

Build a pathway-aware AMR prediction model by combining:
- SNP-level signals from isolate VCF files
- Phenotype labels from CRyPTIC table
- Biological context from KEGG pathways

The notebook implements the full pipeline in one place:
- imports
- data loading
- feature engineering
- heterogeneous graph construction
- model definition
- training and testing
- metrics and interpretation

## Input Data

Required files and folders:
- ./vcf(2024): isolate VCF or VCF.GZ files
- ./CRyPTIC_reuse_table_20240917.csv: phenotype table
- ./kegg_data/tb_drug_knowledge_graph.json: KEGG knowledge graph generated by kegg_pathways.py

Run cells in this order.

1. Cell 1: Imports, constants, and preprocessing utilities

2. Cell 2: Core HGAT graph and utility functions
- Builds base heterogeneous graph with node types:
  - isolate
  - one node type per gene region
- Defines HGATLayer with HeteroConv + GATConv
- Defines masked BCE loss, label masking, threshold tuning, and metrics
- Includes isolate attention extraction helper

3. Cell 3: KEGG integration and pathway-aware model
- Loads KEGG JSON and derives pathway IDs from:
  - pathway_to_genes
  - gene_to_pathways
  - drug_to_pathways
- Adds pathway nodes to the graph
- Adds gene-pathway edges
- Adds isolate-pathway edges based on isolate SNP genes
- Defines PathwayHGAT_AMR model:
  - 2 HGAT layers
  - isolate layer 1 and layer 2 features
  - pathway context attention
  - classifier over concatenated representation
- Defines cross-validation training function
- Defines attribution chain (SNP -> gene -> pathway)

4. Cell 4: Validation and pipeline wrapper
- Defines known mechanism validation for INH, RIF, EMB, PZA
- Summarizes biological overlap hit rates
- Defines reusable end-to-end pipeline function

5. Cell 5: Execute data preprocessing
- Produces prepared inputs:
  - isolate_snp_df
  - phenotype_df
  - snp_embeddings
  - snp_list
  - DRUG_COLS

6. Cell 6: Execute KEGG graph build and model creation
- Loads KEGG knowledge graph

7. Cell 7: Execute training, testing, and metrics
- Runs stratified cross-validation
- Prints per-drug AUROC, AUPRC, sensitivity, specificity, F1, threshold

## Feature Engineering Details

1. Variant filtering:
- Minimum QUAL: 20
- Minimum AF: 0.75
- Keeps simple SNVs in known MTB gene regions

2. SNP embedding:
- Builds isolate x SNP incidence matrix
- Computes SNP co-occurrence matrix
- Converts to positive PMI matrix
- Applies truncated SVD
- Uses embedding dimension 64

3. Label handling:
- Multi-drug labels with NaN allowed for missing outcomes
- Masked loss and masked metrics ensure missing labels are ignored

## What KEGG Adds in This Implementation

The implemented model does not treat KEGG as a post-hoc annotation table. KEGG is represented as explicit graph structure used during message passing.

Implemented KEGG objects used by the model:
- pathway entities: each KEGG pathway ID becomes a pathway node
- gene-to-pathway knowledge: used to connect MTB resistance genes to pathway nodes
- drug-to-pathway knowledge: used to ensure pathway universe is biologically relevant to TB drug mechanisms

Implemented biological constraints in the graph:
- resistance-associated genes that do not map to KEGG pathways remain in SNP-level HGAT channels
- genes with KEGG mappings are lifted into pathway space through gene-pathway edges
- each isolate is linked to pathways it likely perturbs, based on its observed mutated genes

In short, KEGG is integrated as a mechanistic layer between gene-level variation and drug-resistance prediction, not as metadata.

## Graph Design (Implemented Semantics)

The heterogeneous graph is implemented with the following node sets and relations.

Node types:
- isolate: one node per sample, feature = pooled SNP embedding profile
- gene-specific SNP node types: one node type per resistance gene region (for example rpoB, katG, embB)
- pathway: one node per KEGG pathway ID
- gene: compact gene-index node type used only for explicit gene-pathway connectivity

Edge types and biological meaning:
- isolate -> has_gene -> gene-specific SNP nodes:
  - encodes which SNP events are present in each isolate for each gene region
- gene-specific SNP -> in_gene -> isolate:
  - reverse channel for bidirectional message flow
- gene -> participates_in -> pathway:
  - encodes curated KEGG gene membership
- pathway -> includes_gene -> gene:
  - reverse channel for pathway-to-gene propagation
- isolate -> in_pathway -> pathway:
  - encodes isolate-level pathway involvement inferred from mutated mapped genes
- pathway -> has_isolate -> isolate:
  - reverse channel for pathway-to-isolate propagation

Why this matters for predictions:
- isolate risk signal is informed by both direct SNP evidence and pathway-level context
- two isolates with different SNPs can still share pathway-level mechanism signal
- pathway context helps regularize sparse SNP evidence with biological structure

## Model Design

PathwayHGAT_AMR:
- Two HGAT layers over full heterogeneous graph
- Isolate representation from both layers (iso_l1 and iso_l2)
- Pathway context computed by isolate-to-pathway attention
- Final classifier uses concatenated [iso_l1, iso_l2, pathway_context]

Implemented prediction representation:
- iso_l1 captures first-hop neighborhood effects (local SNP/gene relations)
- iso_l2 captures deeper relational effects including pathway-mediated propagation
- pathway_context is an isolate-specific weighted summary over pathway nodes
- final logits are produced from the joint representation [iso_l1, iso_l2, pathway_context]

This means pathway influence is part of the forward pass used to predict resistance probabilities, not only used in interpretation.

## Training and Evaluation

Training:
- Optimizer: Adam
- Gradient clipping enabled
- Masked BCE with optional positive class weights
- Cross-validation over isolates

Evaluation metrics (per drug):
- AUROC
- AUPRC
- Sensitivity
- Specificity
- F1
- Tuned threshold

## Biological Interpretation

Attribution pipeline:
- Extract isolate-specific SNP attentions
- Aggregate to gene-level scores
- Project to pathway-level scores using KEGG mapping

Implemented attribution chain:
- SNP level: which mutations the model attends to for a chosen isolate
- Gene level: aggregated SNP attention per resistance gene
- Pathway level: KEGG-projected mechanism scores derived from gene attention

This gives a direct biological narrative: mutation evidence -> gene mechanism -> pathway mechanism.

Validation:
- Compares top attributed genes with known resistance mechanisms
- Reports overlap and hit rate per drug
